Claude Code Harness Full

Claude Code のスキル・エージェント・設定テンプレートをまとめた構成キットです。
新規プロジェクトにデプロイすることで、Claude Code の機能をすぐに使える状態にします。

ディレクトリ構成

.
├── project-config/     # プロジェクトにデプロイされる設定
│   ├── skills/         # スキル群（スライド生成、レポート等）
│   └── agents/         # エージェント定義
├── global-config/      # ~/.claude/ に配置するグローバル設定
│   ├── skills/         # グローバルスキル
│   └── agents/         # グローバルエージェント
├── agents-org/         # マルチエージェント組織シミュレーション
├── docs/               # Claude Code ガイド・ベストプラクティス
├── scripts/            # デプロイスクリプト
├── .env.example        # 環境変数テンプレート
└── .gitignore

セットアップ

1. リポジトリをクローン

git clone https://github.com/Fukuoka-desu/claude-code-harness.git
cd claude-code-harness

2. 環境変数を設定

cp .env.example .env

.env を編集して以下を設定: - GEMINI_API_KEY — Gemini API キー（管理者からDMで受け取ってください） - DEPLOY_TARGETS — デプロイ先プロジェクトのパス

3. プロジェクトにデプロイ

# 単体デプロイ
./scripts/deploy-to-project.sh ~/your-project

# .env に定義した全プロジェクトに一括デプロイ
./scripts/deploy-all.sh

デプロイされるもの: - .claude/skills/ — スキル群 - .claude/agents/ — エージェント定義（nanobanana SubAgent 含む） - .mcp.json — MCP サーバー設定

4. Gemini API キーをプロジェクトに配置

デプロイ先プロジェクトのルートに .env.local を作成:

echo "GEMINI_API_KEY=your_key_here" > ~/your-project/.env.local

含まれるスキル

カテゴリ スキル例
スライド生成 業種別提案（美容/介護/建設/製造/運送/福祉等）、代理店経由（電通/博報堂）、AI導入提案
ドキュメント 企画書、セールス提案、運用レポート、継続打診資料、効果測定分析
動画 スライド→動画変換、画面収録→マニュアル生成

必要環境

  • Claude Code CLI
  • Python 3.10+
  • Gemini API Key
  • LibreOffice（動画・PDF変換スキル使用時）

Auto Mode（任意）

長いタスクで承認プロンプトを減らす Auto mode を使う場合、起動時に次のようにする（Team / Enterprise / API プラン、Sonnet 4.6 または Opus 4.6 が前提）。

claude --enable-auto-mode

初めから Auto で始める場合は claude --enable-auto-mode --permission-mode auto。詳細は docs/claude-code-complete-guide.md の「Auto Mode を有効にする」を参照。
