Portal / 部門別ツール / Project OS / AI連携

まずPortalで探す。タスクはProject OSで見て、必要な作業だけAIに渡す。

Portalは全ツールの入口です。検索バー下の「部門別に探す」から担当部門マスターを開くと、MM、SM、CC、CS、TSPなど部門ごとの関連ツールだけに絞れます。AI連携は「AIで使う」「AI要約」「AI作業に渡す」の3つだけ覚えれば使えます。

Portalでツールを探す

Portalでは全ツールを一覧で見られます。数が多いと迷うので、最初に検索か部門別表示で絞ります。

検索する

ツール名や業務名が分かるときは、上部の検索欄で直接探します。

部門別に探す

検索バー下の「部門別に探す」から担当部門マスターを開き、MM、SM、CC、CS、TSPなどを選びます。

AIで使う

AI連携できるものだけ見たいときは「AIで使う」表示に切り替えます。

やりたいこと押す場所何が起きるか
担当部門のツールだけ見たい部門別に探す / 担当部門マスター選んだ部門に関連するカードだけが表示されます。
AIに渡す前に内容を確認したいAI要約そのツールで何を確認し、AIに何を任せるかが短く表示されます。
Claude CodeやCursorで作業したいAI作業に渡す選んだツールの目的、URL、補足情報をAI作業用に渡します。
AIに渡すのは必要なカードだけです。
迷ったら、まず「AI要約」を読んでから「AI作業に渡す」を押します。MCPや内部ツール名を覚える必要はありません。

毎朝の基本フロー

画面を見る順番と、Claude Codeへ移るタイミングを固定すると迷いません。

1個人ページを開く/project-os/me を開きます。
2ボードを見る未着手、進行中、判断待ち、停止、完了を確認します。
3ガントを見る期限が近いもの、詰まりそうなものを見ます。
4今日やること右側ボタンから一覧で確認します。
5/taskで作業CursorのClaude Codeからタスクを選びます。
6進捗更新80%、確認待ち、完了などを更新します。
7申し送り次に見る人が分かるコメントを残します。

個人ボードの使い方

個人ボードはトップ画面の一番上に横一列で表示されます。カードは必要最低限の情報だけを出し、詳細はクリックして開きます。

未着手

まだ始めていないタスク。着手したら進行中へ動かします。

進行中

自分が今進めているタスク。進捗率をこまめに更新します。

判断が必要

確認、判断、承認、レビューが必要な状態です。

止まっている

依存関係、不明点、素材待ちなどで止まっている状態です。

完了

作業が終わった状態です。進捗率は100%になります。

ドラッグで状態変更できます。
例: 作業開始なら「未着手」から「進行中」へ、レビュー待ちなら「判断が必要」へ動かします。

AI連携の使い方

AI連携は、Portalのツール情報やProject OSのタスク情報をClaude Code / Cursorへ渡して、調査、整理、実装、文章作成を進めるための入口です。普段は「AI要約」と「AI作業に渡す」を使います。

Portalから渡す

使いたいツールカードで「AI要約」を確認し、「AI作業に渡す」を押します。

タスクから渡す

Project OSで作業タスクを選び、必要ならClaude Code / Cursorで /task を実行します。

結果を戻す

作業後は状態、進捗率、申し送りをProject OSへ戻します。

個人のClaudeログイントークンはTrePro OSに保存しません。
社員のMac上のClaude Code / Cursorが、必要な情報だけを受け取って作業します。MCPという言葉を覚えなくても、Portalでは「AI作業に渡す」、Project OSでは /task接続 を使えば十分です。

MCPでできること

MCPは、Claude CodeやCursorがProject OSと安全にやり取りするための裏側の仕組みです。利用者が普段見る名前ではないので、操作としては「AI作業に渡す」と /task接続 だけ覚えれば問題ありません。

MCP tool内容
get_today_tasks今日やるタスクを取得
get_my_tasks自分のタスクを取得
get_task_detailタスク詳細を取得
get_task_commentsコメントや履歴を取得
add_task_comment申し送りコメントを追加
update_task_status状態を変更
update_progress進捗率を変更
create_taskタスクを作成
create_tasks_from_meeting会議内容からタスクを作成
record_external_progress外部進捗を記録
画面上の言葉に置き換えると簡単です。
MCP toolはAI側の内部操作名です。社員向けには「タスクを読む」「コメントを追加する」「進捗を更新する」と理解すれば十分です。

外部から取り込み

すでにスプレッドシートやNotionで管理しているタスクは、ヘッダーの「外部から取り込み」から入れます。

取り込み元使い方
CSV / 貼り付け表データをコピーして貼り付ける
Google Sheets公開CSV URL、またはシート内容を貼り付ける
NotionCSVエクスポート、または表を貼り付ける
プレビュー確認までは反映されません。
追加可能、重複、無効行を確認して、選んだ行だけ確定します。

会議・チャット・現場報告の進捗同期

Claude Code上の作業だけでなく、Slack、Chatwork、tl;dv、Meet、Zoom、Teams、撮影現場などの報告も進捗に反映できます。

POST /api/project-os/external-progress
Authorization: Bearer <token>
Content-Type: application/json
{
  "sourceType": "slack",
  "taskId": "task_id",
  "content": "Slackで実装完了報告あり。レビュー待ちに移行。",
  "status": "確認待ち",
  "progress": 80,
  "evidenceUrl": "https://example.com/thread",
  "occurredAt": "2026-06-27T15:30:00+09:00"
}
未認証POSTは拒否されます。
SlackやChatworkなどから自動同期する場合も、Bearer tokenまたはProject OSセッションが必要です。
