# TrePro OS Portal・Project OS使い方ガイド

更新日: 2026-06-29  
対象: Portal / 部門別ツール / Project OS / 個人タスクボード / Claude Code・Cursor連携 / MCP / 外部進捗同期

## まず何をするツールか

TrePro OSは、Portalで必要なツールを探し、Project OSで個人のタスクとチーム進捗を見て、必要な作業だけClaude CodeやCursorへ渡すための入口です。

Portalでは全ツールを一覧で見られます。  
数が多いと迷うので、まず検索するか、検索バー下の「部門別に探す」から担当部門マスターを開きます。MM、SM、CC、CS、TSPなどを選ぶと、部門ごとの関連ツールだけに絞れます。

AI連携は「AIで使う」「AI要約」「AI作業に渡す」の3つだけ覚えれば使えます。

- AIで使う: AI連携できるツールだけを見る
- AI要約: そのツールをAIにどう使わせるか確認する
- AI作業に渡す: Claude CodeやCursorで続きの作業を始める

毎朝見る場所は `/project-os/me` です。  
開発作業を進める人は、初回だけ `/task接続` を押します。次回以降はCursorやClaude Code側で `/task` と入力すれば、Project OSのタスク一覧から作業を始められます。

## Portalでツールを探す

Portalでの基本は以下です。

| やりたいこと | 押す場所 | 何が起きるか |
| --- | --- | --- |
| ツール名や業務名で探したい | 検索欄 | 該当するツールカードだけが残る |
| 担当部門のツールだけ見たい | 部門別に探す / 担当部門マスター | MM、SM、CC、CS、TSPなど部門ごとのカードに絞れる |
| AIに渡せるツールだけ見たい | AIで使う | AI連携できるカードだけを確認できる |
| AIに渡す前に内容を確認したい | AI要約 | 目的、見るポイント、AIに任せる作業が短く分かる |
| Claude CodeやCursorで作業したい | AI作業に渡す | 選んだツールの目的、URL、補足情報をAI作業へ渡す |

AIに渡すのは必要なカードだけです。  
迷ったら、まず「AI要約」を読んでから「AI作業に渡す」を押します。MCPや内部ツール名を覚える必要はありません。

## 毎朝の基本フロー

1. Portalを開き、必要なツールを検索または部門別に探す
2. Project OSの個人ページを開く
3. 個人ボードで今日の詰まりを見る
4. ガントで期限が近いものを見る
5. 右側の「今日やること一覧」を開いて、今日やる作業を確認する
6. AIに任せる作業なら「AI作業に渡す」またはCursor / Claude Codeの `/task` を使う
7. 作業が進んだら、Claude Code上またはProject OS画面上で進捗率と状態を更新する
8. 必要なら申し送りコメントを残す

## 個人ボードの使い方

個人ボードはトップ画面の最上部にあります。  
横一列で、状態ごとにタスクが並びます。

状態は以下です。

| 状態 | 使う場面 |
| --- | --- |
| 未着手 | まだ作業を始めていない |
| 進行中 | 自分が今進めている |
| 判断が必要 | 誰かの確認、判断、承認が必要 |
| 止まっている | 依存関係や不明点で止まっている |
| 完了 | 作業が終わった |

タスクカードはドラッグで列を移動できます。  
例えば、作業を始めたら「未着手」から「進行中」へ動かします。確認待ちになったら「判断が必要」へ動かします。

カードに表示する情報は最低限です。詳しく見るときはカードをクリックして、タスク詳細を開きます。

## タスク追加の使い方

トップヘッダーの「タスク追加」または各列の「+追加」からタスクを作れます。

入力する項目は以下です。

| 項目 | 入力内容 |
| --- | --- |
| タスク名 | 何をやるか |
| 親プロジェクト | どのPJに紐づくか |
| 部門/ジャンル | CC、MM、SM、CS、TSP、AIなど |
| 状態 | 未着手、進行中、確認待ち、完了など |
| タスク種別 | 恒久運用、短期、中期、長期 |
| 進捗率 | 0から100% |
| 期限 | いつまでにやるか |
| 優先度 | 高、中、低 |
| 売上直結度 | 高、中、低 |

完了にすると進捗率は100%になります。  
途中経過は25%、50%、75%、80%などで更新できます。

## タスク詳細の使い方

タスクカードをクリックすると詳細が開きます。

上側では手動入力をします。

- 状態を変更する
- 進捗率を変更する
- ジャンルを変更する
- 期限を変更する
- 優先度を変更する
- 成果物や完了条件を書く
- 止まっている理由を書く

下側にはClaude Code / Cursor連携があります。  
手動で状況を整理してから、必要なときだけAI連携を使う構成です。

## AI連携の使い方

AI連携は、Portalのツール情報やProject OSのタスク情報をClaude Code / Cursorへ渡して、調査、整理、実装、文章作成を進めるための入口です。

最初に覚える操作は以下です。

| 画面 | 押すもの | 使う場面 |
| --- | --- | --- |
| Portal | AIで使う | AI連携できるツールだけ見たいとき |
| Portal | AI要約 | そのツールをAIにどう使わせるか確認したいとき |
| Portal | AI作業に渡す | Claude Code / Cursorで続きの作業をしたいとき |
| Project OS | `/task接続` | 初回だけ、CursorとClaude Codeからタスクを扱えるようにするとき |
| Claude Code | `/task` | 自分のタスク一覧から作業を始めたいとき |

### 初回だけやること

Project OSの個人ページで `/task接続` を押します。

これで以下が設定されます。

- CursorからProject OSのタスクを読めるようにする
- Claude Codeで `/task` が使えるようにする
- MCP経由でタスクの状態や進捗を更新できるようにする
- 個人のClaudeログイントークンはTrePro OSに保存しない

社員はターミナル操作をする必要はありません。  
ボタンを押すと、裏側でCursorとClaude Codeの設定を行います。

Portalから作業を始める場合は、対象ツールカードの「AI要約」を読んでから「AI作業に渡す」を押します。

### 次回以降の使い方

Cursor上のClaude Codeで以下を入力します。

```text
/task
```

すると、Project OSのタスク一覧が出ます。  
作業したいタスクを選ぶと、タスク詳細、コメント、完了条件を読んだうえで作業を始められます。

### Claude Code上で頼めること

```text
今日の自分のタスクを出して
```

```text
このタスクの詳細と過去コメントを読んで、作業計画を作って
```

```text
Portalで渡したツールの目的を整理して、今日やる手順にして
```

```text
このタスクを進捗80%にして、状態を判断が必要に変更して
```

```text
レビュー待ちになったので、申し送りコメントを追加して
```

```text
完了したのでProject OS側も完了にして、進捗100%にして
```

## MCPで裏側で動いていること

MCPは、Claude CodeやCursorがProject OSと安全にやり取りするための裏側の仕組みです。利用者が普段見る名前ではありません。

操作としては、Portalでは「AI作業に渡す」、Project OSでは `/task接続` だけ覚えれば問題ありません。

主な機能は以下です。

| MCP tool | 内容 |
| --- | --- |
| get_today_tasks | 今日やるタスクを取得 |
| get_my_tasks | 自分のタスクを取得 |
| get_task_detail | タスク詳細を取得 |
| get_task_comments | コメントや履歴を取得 |
| add_task_comment | 申し送りコメントを追加 |
| update_task_status | 状態を変更 |
| update_progress | 進捗率を変更 |
| create_task | タスクを作成 |
| create_tasks_from_meeting | 会議内容からタスクを作成 |
| record_external_progress | 外部進捗を記録 |

画面上の言葉に置き換えると、MCP toolは「タスクを読む」「コメントを追加する」「進捗を更新する」という内部操作名です。

## 外部から取り込みの使い方

すでにスプレッドシートやNotionで管理しているタスクは、ヘッダーの「外部から取り込み」から入れられます。

対応している取り込み元は以下です。

| 取り込み元 | 使い方 |
| --- | --- |
| CSV / 貼り付け | 表データをコピーして貼り付ける |
| Google Sheets | 公開CSV URL、またはシート内容を貼り付ける |
| Notion | CSVエクスポート、または表を貼り付ける |

取り込み時は、いきなり反映されません。  
プレビューで重複や無効行を確認して、選んだ行だけ確定します。

## 会議・チャット・現場報告の進捗同期

Project OSは、Claude Code以外の進捗も受け取れるようにしています。

想定している進捗元は以下です。

- Slack
- Chatwork
- tl;dv
- Google Meet
- Zoom
- Teams
- 撮影現場などのオフライン活動

外部ツールから同期する場合は、認証付きAPIに進捗イベントを送ります。

```http
POST /api/project-os/external-progress
Authorization: Bearer <token>
Content-Type: application/json
```

```json
{
  "sourceType": "slack",
  "taskId": "task_id",
  "content": "Slackで実装完了報告あり。レビュー待ちに移行。",
  "status": "確認待ち",
  "progress": 80,
  "evidenceUrl": "https://example.com/thread",
  "occurredAt": "2026-06-27T15:30:00+09:00"
}
```

未認証の投稿は拒否されます。  
そのため、SlackやChatworkなどから自動同期する場合も、Bearer tokenまたはProject OSセッションが必要です。

## 管理者画面の使い方

管理者は `/admin/project-os` を使います。

できることは以下です。

- 全社プロジェクトを見る
- 全社タスクを見る
- メンバー一覧を見る
- メンバーの個人ページを開く
- プロジェクトを作成、編集、削除、非表示、復元する
- タスクを作成、編集、削除する
- AIアラートを承認または却下する
- 責任者別、部門別、ガントで進捗を見る
- 組織ツリーを確認、編集する

AIアラートは、自動で勝手に反映されません。  
管理者が内容を確認して、承認したものだけが反映されます。

## 困ったとき

| 症状 | 見る場所 |
| --- | --- |
| Cursorが開かない | `/task接続` の初回設定、Cursorがインストール済みか確認 |
| `/task` でタスクが出ない | MCP接続、ローカルトークン期限、ログイン状態を確認 |
| 担当部門マスターが見つからない | Portalの検索バー下にある「部門別に探す」を確認 |
| AI連携で何を押せばいいか分からない | まず「AIで使う」に切り替え、カードの「AI要約」を読んでから「AI作業に渡す」を押す |
| 進捗が反映されない | Claude Code側で `update_progress` または `update_task_status` が実行されたか確認 |
| 外部進捗が反映されない | 認証、taskId、sourceType、status、progressを確認 |
| どのタスクをやるべきか分からない | 今日やること一覧、ガント、通知を見る |
| タスクが多すぎる | 種別、ジャンル、期間で絞り込む |

## 一番大事な使い方

まずPortalで必要なツールを探します。  
担当部門ごとに見るときは「部門別に探す」から担当部門マスターを開きます。  
AIに渡すときは「AI要約」を読んでから「AI作業に渡す」を押します。  
毎朝はProject OSで全体を見ます。  
作業タスクはCursor / Claude Codeの `/task` から始められます。  
進捗、状態、申し送りはProject OSへ戻します。  

この流れが定着すると、Portalのツール、個人ボード、チーム進捗、Claude Code上の作業、会議やチャットの報告がひとつにつながります。
