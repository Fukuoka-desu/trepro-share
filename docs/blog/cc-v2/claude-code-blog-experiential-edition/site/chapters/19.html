第19章

自分の口癖を、AIに覚えさせる Claude Codeを自分専用に育てる

『他人の Skill をコピーしてきたけれど、自分の仕事には合わない』。 これは品質の問題ではなく、自分の口癖を AI に教えていないだけです。

ほとんどの人は、AIを『汎用ツール』として使っています。 ですが、AIは、何を好み・何を避け・どこを丁寧に確認したいかという、あなた固有の癖を覚えることもできます。

Auto Memory・User Rules・個人用 Skill は、自分の作業様式を AI に学習させる仕掛けです。 毎回の手直しが減り、最初から自分の文体・形式で返ってくるようになります。

この章を読み終わる頃には、AI が『自分のために働く道具』として馴染んできます。

この章の概観 ／ 何を扱い、何ができるか

この章で扱う機能と、それで何ができるようになるか。実装に入る前に、全体像をここでつかみます。

  • Privacy by Design（保存しないものを決める）

    秘密・個人情報・顧客データ・雑談を除外し、保存対象を関心・学習課題・好む形式に限定するパーソナライズ設計。

    入力
    会話ログ、扱う情報のセンシティビティ
    出力
    保存対象/除外対象を分けた抽出ルール

  • Interest Profile Skill

    未分析ログから関心・学習課題・好む形式だけを抽出してINTERESTS.mdへまとめるSkill。本人が編集可能。

    入力
    セッションログ、過去会話
    出力
    編集可能なINTERESTS.mdプロファイル

  • Recency Score（重み付け）

    新しさと深掘り回数で重み付けし、現在の関心を反映するスコアリング。古い関心と現在を混ぜない。

    入力
    抽出済み関心信号、時間軸
    出力
    現在優先のスコア順関心リスト

  • 差分処理（処理済み位置の管理）

    処理済み位置を保持し、毎回全ログを再走査しない実装パターン。処理時間と費用を構造的に抑える。

    入力
    新規ログと前回処理位置
    出力
    差分のみ処理した抽出結果

  • 学習問題生成

    Profileを元に調査テーマや復習問題を作る出力機能。学んだことを定着させるための再利用。

    入力
    INTERESTS.md、苦手分野
    出力
    復習問題、深掘り調査テーマ

実装リファレンス

19.1 外部化する情報

なぜここを読むか AIに何を覚えさせ、何を覚えさせないかをまず外から見える形にしないと、パーソナライズは個人情報の暴露と紙一重になります。

やる／やらないとどうなるか 関心・詰まる概念・好む形式といった再利用価値のある信号だけをファイル化し、本人がいつでも編集・削除できる構造になります。

  • 最近の関心。
  • よく詰まる概念。
  • 好む説明形式。
  • 継続中の学習Theme。
  • 次のResearch候補。
  • 復習すべき内容。

これは何か 個人化用ディレクトリの構成例で、状態・信号・プロファイル・学習・好み・問題を1か所に集約しています。

実行するとどうなるか 保存対象と管理対象が見える形になり、本人がレビュー・削除できる範囲が物理的に1フォルダで完結します。

それぞれの意味

state.json
前回どこまで処理したかを覚えておくチェックポイントです。差分追記の起点になり、毎回全ログ再走査による費用爆発を防ぎます。
signals.jsonl
ノイズと機密を落とした関心信号を1行1件で追記する原本です。プロファイル化の素材で、ここを消せば学習痕跡も消えます。
INTERESTS.md
現在の関心トピックを本人が読める形にまとめたプロファイルです。Skillが更新し、本人が手で削除・修正できる前提のファイルです。
LEARNING.md
詰まっている概念や継続中の学習テーマを記録する場所です。復習問題生成や次のResearch候補の供給元として再利用します。
preferences.md
好む説明形式・避けたい言い回しを書き留める好みファイルです。AIが返答スタイルを毎回合わせ直す手間がここで解消します。
quizzes/
LEARNING.mdから生成した復習HTMLを置くフォルダです。想起練習をローカルだけで回せ、外部送信なしの学習ループを成立させます。
personalization/
├── state.json
├── signals.jsonl
├── INTERESTS.md
├── LEARNING.md
├── preferences.md
└── quizzes/

19.2 Privacy by Design

なぜここを読むか 個人化を進めるほど、顧客情報や評価につながる属性まで吸い込む事故が起きやすくなるため、最初に保存禁止の境界を引きます。

やる／やらないとどうなるか ProjectのAllowlistとSecretマスク、人事評価非利用などのルールが先に決まり、後からの『これは消してほしい』を構造的に減らせます。

これは何か Privacy by Designの最低限ルールを箇条書きで宣言する設計メモで、Opt-in方針とユーザー権限を明文化します。

実行するとどうなるか 後から議論になりがちな『これ保存していい？』が事前合意になり、抽出処理側が参照すべき判断軸が固まります。

それぞれの意味

対象ProjectをAllowlist
どのプロジェクトのログだけを学習対象にするかを明示で許可する方式です。書き忘れたプロジェクトは自動的に対象外になります。
顧客案件はOpt-in
顧客情報を含むプロジェクトは本人の明示同意がない限り学習しない原則です。気付かぬうちに顧客データが混入する事故を防ぎます。
SecretをMask
APIキーやトークン文字列を抽出段階で置換して保存しない処理です。後でログ流出しても認証情報まで漏れない構造になります。
生LogのRetention
未加工のログをどのくらい残すかを期間で決めるルールです。残しすぎるほど削除要請への対応コストが膨らみます。
Sensitive Traitを推測しない
健康・思想・性的指向など評価につながる属性を推定対象にしない原則です。差別的判定への利用リスクを構造的に断ちます。
人事評価へ使わない
学習プロファイルを査定や人事評価には使わないという用途限定の宣言です。社員に対する監視ツール化を最初に否定します。
UserがReview/Delete可能
本人がいつでもプロファイルを閲覧・削除できる権限の保証です。納得感が担保され、後からの『これ消して』を構造的に減らします。
- 対象ProjectをAllowlist
- 顧客案件はOpt-in
- SecretをMask
- 生LogのRetention
- Sensitive Traitを推測しない
- 人事評価へ使わない
- UserがReview/Delete可能

19.3 未分析Logだけを抽出するScript

なぜここを読むか 毎回すべてのログを読み直すと処理時間も費用も伸び続けるため、前回処理位置を持つ差分処理が要ります。

やる／やらないとどうなるか ユーザー発話だけを抽出してSecretsをマスクし、ノイズを除いた信号を追記していくJSONLが手に入ります。

Local履歴形式は更新され得る。実環境のJSONL Schemaを確認して使う。

tools/extract_user_signals.py：

#!/usr/bin/env python3
from __future__ import annotations

import argparse
import hashlib
import json
import re
from datetime import datetime, timezone
from pathlib import Path
from typing import Any, Iterable

DEFAULT_ROOT = Path.home() / ".claude" / "projects"
DEFAULT_STATE = Path("personalization/state.json")
DEFAULT_OUTPUT = Path("personalization/signals.jsonl")

NOISE = [
    re.compile(r"^(こんにちは|ありがとう|了解|ok|okay|yes|no)[。!！ ]*$", re.I),
    re.compile(r"^/[a-z0-9_-]+(?:\s+.*)?$", re.I),
]
SECRETS = [
    re.compile(r"(?i)(api[_-]?key|token|secret|password)\s*[:=]\s*[^\s,;]+"),
    re.compile(r"\b(?:sk|ghp|github_pat)_[A-Za-z0-9_-]{16,}\b"),
]


def load_json(path: Path, default: Any) -> Any:
    if not path.exists():
        return default
    try:
        return json.loads(path.read_text(encoding="utf-8"))
    except (json.JSONDecodeError, OSError):
        return default


def atomic_json(path: Path, value: Any) -> None:
    path.parent.mkdir(parents=True, exist_ok=True)
    tmp = path.with_suffix(path.suffix + ".tmp")
    tmp.write_text(json.dumps(value, ensure_ascii=False, indent=2), encoding="utf-8")
    tmp.replace(path)


def text_blocks(content: Any) -> Iterable[str]:
    if isinstance(content, str):
        yield content
    elif isinstance(content, list):
        for block in content:
            if isinstance(block, str):
                yield block
            elif isinstance(block, dict) and block.get("type") in {"text", "input_text"}:
                value = block.get("text") or block.get("content")
                if isinstance(value, str):
                    yield value


def user_text(record: dict[str, Any]) -> str:
    record_type = str(record.get("type", "")).lower()
    message = record.get("message")
    if isinstance(message, dict):
        role = str(message.get("role", "")).lower()
        if role != "user" and record_type != "user":
            return ""
        content = message.get("content", "")
    else:
        role = str(record.get("role", "")).lower()
        if role != "user" and record_type != "user":
            return ""
        content = record.get("content", "")
    return "\n".join(x.strip() for x in text_blocks(content) if x.strip()).strip()


def redact(text: str) -> str:
    for pattern in SECRETS:
        text = pattern.sub("[REDACTED_SECRET]", text)
    return text[:5000]


def is_noise(text: str) -> bool:
    normalized = " ".join(text.split()).strip()
    return len(normalized) < 12 or any(p.match(normalized) for p in NOISE)


def source_id(path: Path, root: Path) -> str:
    rel = str(path.relative_to(root))
    return hashlib.sha256(rel.encode()).hexdigest()[:16]


def main() -> int:
    parser = argparse.ArgumentParser()
    parser.add_argument("--log-root", type=Path, default=DEFAULT_ROOT)
    parser.add_argument("--state", type=Path, default=DEFAULT_STATE)
    parser.add_argument("--output", type=Path, default=DEFAULT_OUTPUT)
    parser.add_argument("--allow-project", action="append", default=[])
    args = parser.parse_args()

    root = args.log_root.expanduser().resolve()
    if not root.exists():
        raise SystemExit(f"log directory not found: {root}")

    state: dict[str, int] = load_json(args.state, {})
    new_state = dict(state)
    allow = {x.lower() for x in args.allow_project}
    rows: list[dict[str, Any]] = []

    for path in sorted(root.rglob("*.jsonl")):
        rel = str(path.relative_to(root)).lower()
        if allow and not any(name in rel for name in allow):
            continue
        lines = path.read_text(encoding="utf-8", errors="replace").splitlines()
        start = min(max(0, int(state.get(str(path), 0))), len(lines))
        for line_no, line in enumerate(lines[start:], start=start + 1):
            try:
                record = json.loads(line)
            except json.JSONDecodeError:
                continue
            if not isinstance(record, dict):
                continue
            text = redact(user_text(record))
            if not text or is_noise(text):
                continue
            timestamp = record.get("timestamp") or record.get("created_at")
            if not isinstance(timestamp, str):
                timestamp = datetime.now(timezone.utc).isoformat()
            sid = source_id(path, root)
            rid = hashlib.sha256(f"{sid}:{line_no}:{text}".encode()).hexdigest()
            rows.append({"id": rid, "timestamp": timestamp, "source": sid, "text": text})
        new_state[str(path)] = len(lines)

    args.output.parent.mkdir(parents=True, exist_ok=True)
    with args.output.open("a", encoding="utf-8") as handle:
        for row in rows:
            handle.write(json.dumps(row, ensure_ascii=False) + "\n")
    atomic_json(args.state, new_state)
    print(f"appended={len(rows)} output={args.output}")
    return 0


if __name__ == "__main__":
    raise SystemExit(main())

これは何か ユーザー発話だけを抽出し、Secretsをマスクして差分追記するPythonスクリプトで、state.jsonに処理済み行を覚えます。

実行するとどうなるか 毎回全ログを読み直さずに信号JSONLが追記され、ノイズと機密が落ちた状態でプロファイル化の素材が手に入ります。

mkdir -p personalization tools
python3 tools/extract_user_signals.py --allow-project my-workspace

19.4 Interest Profile Skill

なぜここを読むか 抽出した信号をそのまま渡しても、本人が理解できる形に整っていないと活用も削除もできません。

やる／やらないとどうなるか INTERESTS.mdとLEARNING.mdを更新するSkillが、差分付きで本人レビュー可能な形に変換してくれます。

---
name: interest-profile
description: Analyze new user-authored interaction signals and update a concise privacy-safe interest and learning profile. Use when syncing interests, personalizing research, or creating review material.
argument-hint: "sync | research-brief | quiz"
disable-model-invocation: true
---

# Objective
Turn `personalization/signals.jsonl` into user-reviewable context.
Never infer sensitive traits or employee performance.

# Exclusions
Do not retain credentials, customer secrets, health, political, religious,
sexual, financial-account, or private family information.
Do not quote long messages.

# Scoring
Assess frequency across sessions, follow-up depth, explicit interest,
recency, and whether the topic remains active.

# Workflow
1. Read new signals and existing profiles.
2. Remove greetings, commands, pasted source text, and operational noise.
3. Group similar topics.
4. Merge rather than duplicate.
5. Mark uncertain inferences as hypotheses.
6. Keep no more than 15 active topics.
7. Update INTERESTS.md and LEARNING.md.
8. Show the diff for user approval.

19.5 Recency Score

なぜここを読むか 古い興味と現在の興味を同じ重みで扱うと、すでに関心が薄れた話題が出続けるため、新しさで割り引く設計が要ります。

やる／やらないとどうなるか 頻度と深掘りに半減期を掛けた数値で並び替えでき、現在の関心が上に出るプロファイルになります。

これは何か 明示的関心・セッション数・深掘り・繰り返しに重みを置き、最後に新しさの半減期で割り引くスコア式です。

実行するとどうなるか 古い関心と現在の関心を分離した順位が得られ、人間評価ではなく並び替え補助としてだけ使う前提が共有されます。

それぞれの意味

explicit_interest
本人が明示的に『興味がある』と発言した回数です。重み3で最も強く効かせ、推測ベースの関心と明示の関心を分けて扱います。
distinct_sessions
別セッションで同じ話題が出てきた回数です。重み2を置き、一日だけの瞬間的熱と継続的関心を切り分けます。
follow_up_depth
そのトピックを何段階深掘りしたかの深さです。表面的に触れただけの話題と、踏み込んで質問した話題を区別する指標になります。
repeated_actions
同じ操作や調査を繰り返した回数です。発言にならない行動レベルの関心まで含め、口に出さない興味を拾える指標になります。
0.5^(days_since_last / half_life_days)
最後に触れてからの経過日数で半減させる時間減衰係数です。古い興味と現在の興味を同じ重みで扱う事故を防ぎます。
base = explicit_interest × 3
     + distinct_sessions × 2
     + follow_up_depth
     + repeated_actions
score = base × 0.5^(days_since_last / half_life_days)

数値は並べ替え補助であり、人間評価へ使わない。

19.6 学習問題

なぜここを読むか ためた関心や苦手分野は、読み返すだけでは定着しないので、想起練習に再利用する仕組みが要ります。

やる／やらないとどうなるか LEARNING.mdから自信度と再出題を持つ復習HTMLが生成され、ローカルだけで完結する学習ループが回せます。

`LEARNING.md`から想起学習用HTMLを作る。
20問。定義、比較、原因、実装、Debugを混ぜる。
答えを初期非表示にし、自信度と不正解再出題を持つ。
Local Storageには回答状態だけを保存する。

19.7 音声・画像・割り込み

なぜここを読むか 音声入力は便利ですが、Command・Path・数値の誤認識のまま実行すると個人化の利便性が事故に変わります。

やる／やらないとどうなるか 実行前に目的・対象・禁止・完了条件を復唱させる運用に切り替え、画像はSecretsをマスクして使えるようになります。

長い依頼は音声入力と相性がよいが、Command、Path、数値を実行前に復唱させる。

これは何か 音声入力時にAIへ復唱を要求する短い指示文で、Command・Path・数値の聞き取り誤差を実行前にあぶり出します。

実行するとどうなるか 誤認識のまま処理が走るリスクが減り、実行前に修正可能なゲートが1段挟まる運用に変わります。

それぞれの意味

目的
何を達成したい依頼なのかをAIに言い直させる項目です。音声の聞き間違いで作業対象がずれたまま走るリスクを実行前に潰せます。
対象File
どのファイル・パスに対する作業かを復唱させる項目です。Path文字列の誤認識を、実行前のテキスト確認で必ず一回挟めます。
禁止
してはいけない操作を明示で言わせる項目です。削除・送信・公開などの暴発を、AI側からも禁止条件として復唱させて二重ガードにします。
完了条件
どうなったら作業を終えるかを言わせる項目です。曖昧な完了感で走り続けるのを防ぎ、停止条件をAIにも合意させた状態で進めます。
今の依頼を、目的、対象File、禁止、完了条件に分けて復唱してください。まだ実行しないでください。

画像はUI差分やError画面へ使い、SecretsをMaskする。緊急停止は割り込みTextではなく停止操作を使う。

こんな場面で使う

章の内容が実務でどう活きるか。架空の場面・行動・効果の三点で示します。

場面

ある個人ライター（架空）が、AIに『自分専用化したい』と全会話履歴を渡し、結果として顧客個人情報まで学習対象に紛れて怖くなった。

こう使う

保存対象を関心・学習課題・好む形式の三つに限定し、秘密情報・顧客データ・雑談は除外定義を書く。

得られるもの

個人化の便益は得つつ、プライバシーリスクが構造的にゼロへ近づく。

場面

学習担当（架空）が、社員ごとのスキルレベルを把握したいが、毎回全ログを再走査して処理時間が爆発している。

こう使う

処理済み位置と新しさ重みを持つINTERESTS.md抽出Skillを作る。

得られるもの

差分処理になり、関心の最新動向だけ追える。本人が編集・削除できる透明な仕組みになる。
