第18章

AIと2人で、同じ画面を編集する CoworkとLive Artifacts

『AIと一緒に作業しているはずなのに、結局画面を行き来している』。 これは慣れの問題ではなく、共有空間が無いだけです。

ほとんどの人は、AIの出力をコピーし、エディタへ貼り、戻ってきて指示し直しています。 そのたびに、文脈の半分が抜け落ちます。

Cowork と Live Artifacts は、AIと人が同じファイルを同時に編集できる場所です。 AI が直したところがそのまま反映され、人が直したところを AI が拾います。

この章を読み終わる頃には、『AI に頼む』から『AI と並んで書く』へ、距離感が変わります。

この章の概観 ／ 何を扱い、何ができるか

この章で扱う機能と、それで何ができるようになるか。実装に入る前に、全体像をここでつかみます。

  • Cowork（非開発業務の複数ステップ）

    Projectで文脈を固定し、複数ステップの業務処理を継続実行する作業形態。Codeとは別の入口として扱う。

    入力
    繰り返し発生する業務、参照する文脈ファイル
    出力
    継続実行可能なCowork Project

  • Live Artifact 5層構造

    データ取得・整形・表示・AI要約・キャッシュの5層に分けて設計する、開くたびに最新化する業務インターフェース。

    入力
    対象データソース、表示したい指標
    出力
    5層分離されたLive Artifact実装

  • 会議Dashboard Prompt

    Driveの会議ファイルを開くたびに取り込み・要約・キャッシュするダッシュボード用プロンプトのテンプレ。

    入力
    Drive上の会議ファイル群、要約観点
    出力
    リロード時に最新化される会議Dashboard

  • Scheduled設計

    定期実行で重複防止・ロック・ログ・通知・再試行を持たせる業務スケジュール設計。

    入力
    実行頻度、失敗時の挙動、通知先
    出力
    Scheduledジョブ設定と運用ログ

  • Acceptance（更新/キャッシュ/出典）

    Live Artifactの完成判定。更新時刻・キャッシュ振る舞い・出典表示の3点をAcceptance基準にする。

    入力
    完成したArtifact、運用シナリオ
    出力
    3点Acceptance通過記録、Stale時の挙動確認

  • 議事録→週報/Task。
  • Drive資料の整理・要約。

Codeへ移す目安：Git履歴、独自Library/Test、認証/DB、本番Deploy、複雑な監視が必要。

領域 役割
Project Folder、指示、素材
Artifacts 永続的なHTML等
Scheduled 定期実行
Dispatch 離れた端末から指示
Customize Skills/Connectors

UI名より役割を教材化する。

18.3 Live Artifactの5層

Source → Fetch → Transform → Cache → View
Drive     MCP     Parse/Summary  Hash    Filter/Search/Ask

18.4 会議Dashboard Prompt

Google Driveの「全社会議文字起こし」を参照するLive Artifactを作る。

- 最新File一覧
- 日付、題名、参加者、決定、担当、期限
- 全文検索/担当Filter
- 原文Link
- 短いAI要約
- 会議ごとの質問欄
- 失敗/権限不足/空Data状態

Performance:
- source_id + content_hash + prompt_versionでCache
- 変更Fileだけ再要約
- 30秒Timeout、Retry Button

Safety:
- Read-only
- 権限変更なし
- 原文全文/個人情報をLocal Storageへ保存しない

動画で紹介されたArtifact内API名はBetaで変化し得る。作成時の公式Helpを正とする。

18.5 Scheduled設計

- 対象期間
- 既処理判定
- Timezone
- Retry上限
- 重複起動Lock
- 失敗通知
- 保存先
- 0件時の挙動

18.6 Acceptance

[ ] Add/Deleteで更新
[ ] 未変更Dataを再処理しない
[ ] 権限不足説明
[ ] 0件/部分失敗/Timeout
[ ] 原文へ戻れる
[ ] AI要約と原文を区別
[ ] 更新日時
[ ] Mobile
[ ] Write権限不要
[ ] Browser Storageへ機密なし

補助図の制作指示

こんな場面で使う

章の内容が実務でどう活きるか。架空の場面・行動・効果の三点で示します。

場面

ある営業マネージャー（架空）が、週報を毎週手動更新しており、開くたびに最新化されないため週末まで状況が見えない。

こう使う

Driveの会議ファイルを開くたびに取り込み・要約・キャッシュするLive Artifact Dashboardを作る。

得られるもの

開いた瞬間に最新の進捗が見え、週次更新作業そのものが不要になる。

場面

中小オペレーター（架空）が、外部APIを毎リロードで叩く実装にして請求金額がはねた。

こう使う

Cache・更新時刻・出典・失敗時のStale表示を分離して設計する。

得られるもの

コストとUXのバランスが取れ、データ取得失敗時にも古い情報と明示できる。
