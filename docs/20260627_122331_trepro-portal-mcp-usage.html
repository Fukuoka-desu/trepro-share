TrePro Portal MCP 接続設定ガイド

Portalに入っているカード、グループ、Run Handoffの情報を、Codex / Claude Code / Cursor から読み取れるようにするための使い方です。 重要なのは、PortalのボタンだけでIDEへ自動接続するのではなく、Portal側の接続情報をIDE側に設定して接続完了にする、という点です。

Portal MCP 稼働中 read-only Codex / Claude Code / Cursor DB書き込みなし

まず結論

Portal上の MCP接続設定 ボタンは、IDEへ自動で接続するボタンではありません。 IDEに入れる接続情報を表示し、Portal側のMCPが動いているか確認するためのボタンです。

接続の全体像

1

Portal側

MCPサーバーは本番で稼働済みです。PortalのContextを読み取る入口が用意されています。

2

IDE側

Codex / Claude Code / Cursor に、PortalのMCP URLとToken設定を追加します。

3

接続完了

IDE上のAIがPortal情報をPullして、カードやRun Contextを参照できます。

Portalでの操作

  1. Portalを開きます: https://www-treproai.com/portal
  2. 検索欄の右側にある MCP接続設定 を押します。
  3. 接続先IDEとして Codex、Claude Code、Cursor のどれかを選びます。
  4. Portal側を確認 を押します。ここで確認できるのは、Portal MCPがログインsessionで読めるかどうかです。
  5. 表示された設定をコピーし、IDE側に貼り付けます。
注意: 「Portal側を確認」が成功しても、IDE側の設定が完了したわけではありません。 IDE側に設定とTokenを入れて初めて、Codex / Claude Code / Cursorから利用できます。

Codexで使う場合

Portalで Codex を選ぶと、次のような設定が表示されます。 これをCodexのMCP設定に追加します。 

[mcp_servers.trepro_portal]
url = "https://www-treproai.com/api/mcp/portal"
bearer_token_env_var = "TREPRO_PORTAL_MCP_TOKEN"

TokenはHTMLやURLに直接書きません。環境変数として渡します。 

export TREPRO_PORTAL_MCP_TOKEN="管理者から共有されたToken"
Codexから使うときは、Codex側が TREPRO_PORTAL_MCP_TOKEN を読み取り、 Portal MCPへBearer token付きでアクセスします。

Claude Code / Cursorで使う場合

Claude Code

Claude Code側のMCP設定に、Portal MCP URLとBearer tokenを追加します。 Portal画面では接続情報をコピーできます。

Cursor

Cursorで利用するAI拡張またはMCP設定に、同じPortal MCP URLとTokenを設定します。 Cursor Chatへ無理に本文を流し込む設計ではありません。

何が読めるか

Tool 用途 例
search_portal_resources Portalのカードや資料を検索します。 「SM」「会計」「社内マニュアル」などで検索。
get_portal_resource 指定したPortalカードの詳細を取得します。 タイトル、URL、説明、タグ、AI参照可否など。
get_portal_groups Portal上のグループ構成を取得します。 SMグループ、HRグループ、全社マスターなど。
get_run_context Run Handoffで作った実行Contextを取得します。 Portalで組み立てたAI作業内容をIDE側で読む。

安全設計

  • read-only: Portal MCPは読み取り専用です。DBや業務システムへ書き込みません。
  • 本文をURLに載せない: 長いContextや顧客情報をDeep Linkに入れません。
  • Token必須: IDE側から読む場合はBearer tokenが必要です。
  • 権限分離: Portal画面のテストはブラウザsession、IDE側利用はTokenで認証します。

よくある勘違い

勘違い 正しい理解
ボタンを押すだけでCodexに接続される いいえ。ボタンは接続設定を表示します。IDE側への設定追加が必要です。
Portalの情報がURLでIDEへ送られる いいえ。IDE側がMCP APIからPullします。
MCPからDB更新できる いいえ。現在のPortal MCPは読み取り専用です。
TokenをHTMLやURLに貼ってよい いいえ。Tokenは環境変数やIDEの秘密設定で扱います。

社員向けの最短説明

Portalの MCP接続設定 は、AI開発ツールからPortal情報を読めるようにするための設定画面です。 まずPortal側の確認を押し、成功したら表示された設定をCodex / Claude Code / Cursorに入れてください。 設定後、IDE側のAIがPortalのカードやRun Contextを読み取れるようになります。
